Introduction to Artificial Intelligence
Project 2: Development of Medical Diagnosis Expert System
Submission Date: 25th March 2026

Objective:
To design and implement a rule-based expert system that can assist in diagnosing common medical conditions, exemplifying principles of knowledge representation, inference, and user interaction in expert systems.
Students will create an expert system that simulates a medical expert's decision-making for diagnosing simple diseases based on symptoms. The system will use rule-based knowledge representation, forward chaining inference, and a user-friendly interface.

Background and Rationale
•	Understand the concept of expert systems as interactive decision support tools.
•	Study the components: knowledge base, inference engine, user interface.
•	Explore applications in medical diagnosis to grasp real-world relevance.
Learning Outcomes
•	Grasp fundamental concepts of expert systems.
•	Learn rule-based knowledge representation.
•	Implement inference mechanisms (e.g., forward chaining).
•	Develop user interfaces for knowledge-based applications.
•	Evaluate system performance and usability.

Project Phases
Phase 1: Requirements and Planning
•	Identify Diseases and Symptoms: For example, Cold, Flu, and allergies. Symptoms: Fever, cough, sore throat, sneezing, body ache, etc.
•	Define Knowledge Rules: Formulate rules like: IF Fever AND Cough AND Sore Throat THEN Diagnosis is Cold.
Choose Tools and Platforms:
•	Programming Language: Python, Java, or C++
•	Development Environment: IDE of choice (e.g., PyCharm, Eclipse)
Phase 2: Knowledge Base Development
•	Encode the rules derived from medical guidelines.
•	Organize facts about symptoms and conditions.
•	Structure rules logically, ensuring they are non-conflicting.
Phase 3: Inference Engine Implementation
•	Implement forward chaining algorithm:
•	Start with user-provided facts (symptoms).
•	Apply rules to infer possible diagnoses.
•	Continue until no new deductions can be made.
•	Allow the system to suggest possible diagnoses with confidence levels (optional).
Phase 4: User Interface Design
•	Develop a simple command-line or graphical user interface.
Features:
•	Symptom input prompts.
•	Display possible diagnoses and reasoning.
•	Options for additional symptom queries or exit.
Phase 5: Testing and Validation
•	Test with multiple symptom combinations.
•	Validate diagnoses against expected outcomes.
•	Collect user feedback for usability.
Phase 6: Documentation and Reporting
•	Document knowledge rules, system architecture, and inference process.
•	Write user manual and developer notes.
•	Provide system limitations and future improvement suggestions.

Deliverables
•	Source code of the expert system.
•	Knowledge base documentation (rules, facts).
•	User manual explaining how to interact with the system.
•	System report summarizing design decisions, inference mechanism, testing results, and lessons learned.
•	Presentation demonstrating system operation and features.
